# AI-Assisted QA: From Manual Tester to Automation Strategist

> A practical blog series for QA engineers who want to use AI to work smarter, build better automation, and elevate quality across their entire team.

---

## About This Series

This series is written for **manual testers** who want to evolve their craft using AI assistants like [Zen (Zenflow)](https://zenflow.io). No prior automation experience is required. Each phase builds on the last — you can start wherever you are.

The series follows a structured progression from foundational setup all the way to making quality a shared responsibility across the entire team.

---

## Series Overview

| # | Level | What You'll Learn |
|---|---|---|
| [00](./blog/00-preconditions.md) | **Preconditions** | What to prepare before bringing AI into your workflow |
| [01](./blog/01-beginner.md) | **Beginner** | Generate test cases, write bug reports, build coverage maps |
| [02](./blog/02-advanced.md) | **Advanced** | Convert manual tests to Playwright automation with AI |
| [03](./blog/03-expert.md) | **Expert** | Risk strategy, metrics, suite optimization, release decisions |
| [04](./blog/04-ultimate.md) | **Ultimate** | Scale quality across Dev, PO, BA, and DevOps |

---

## Who Is This For?

- **Manual testers** who want to start automating without a coding background
- **Junior QA engineers** looking to level up their skills and strategic impact
- **Mid-level QA engineers** who want to move from execution to strategy
- **QA leads** who want to scale quality across their team and organization

## Tools Referenced

- **[Zen / Zenflow](https://zenflow.io)** — AI assistant for project management and QA workflows
- **[Playwright](https://playwright.dev)** — End-to-end browser automation framework
- **GitHub Actions** — CI/CD pipeline for running automated tests
- Any test management tool (TestRail, Qase, Zephyr, etc.)
